# 设计原则

每一种编程语言都是一组哲学信念的具体表达。1y 并非为了在语法竞赛中胜出而诞生,它诞生于一个朴素的问题:**当我们不再需要为硬件的局限性妥协时,一门函数式语言应当是什么样子?**

现代计算机拥有足够的内存和算力,让我们不必再把整数压缩进 32 位或 64 位,也不必让程序员在每一个并发场景里手动管理锁。然而,绝大多数主流语言仍背负着上世纪硬件约束留下的遗产:定长整数、可变共享状态、函数着色(async 与同步函数的对立)。1y 选择放下这些遗产。

## 1y 为何存在

1y 的起点是两个观察。

第一,**数值计算本不该有溢出**。无论是密码学、金融建模还是组合数学,程序员都反复遭遇同一类 bug:`int32` 溢出、`int64` 溢出、浮点精度丢失。这些不是程序员的疏忽,而是语言设计的缺陷——把硬件的字长泄露给了程序员。1y 从第一天起就把 `Int` 和 `Decimal` 都设计为任意精度,溢出在语言层面被根除。

第二,**并发本不该如此痛苦**。线程 + 锁的组合是数据竞争与死锁的温床;`async/await` 虽然解决了阻塞问题,却引入了函数着色、`Pin`、复杂的 future 生态。1y 选择了另一条路:Actor 模型负责消息传递与隔离状态,软件事务内存(STM)负责共享可变状态。两者都建立在不可变数据之上,让并发代码读起来像顺序代码一样自然。

## 核心设计原则

### 不可变优先(Immutability-first)

在 1y 中,所有值默认不可变。一旦一个值被创建,它就永远不会改变。数据结构是**持久化的**(persistent):对一个列表"添加"元素,不会修改原列表,而是返回一个共享了大部分结构的新列表。

```1y
let xs = [1, 2, 3]
let ys = List.push(xs, 4)
# xs 仍然是 [1, 2, 3],ys 是 [1, 2, 3, 4]
```

这听起来低效,但持久化数据结构通过**结构共享**实现了近乎 O(1) 的复制成本。更重要的是,不可变性从根本上消灭了别名问题:你不需要担心"另一个函数会不会偷偷改掉我的数据"。在并发场景下,这意味着数据可以自由地在 Actor 之间传递,无需拷贝、无需同步。

### 表达式导向(Expression-oriented)

1y 是一门表达式语言:几乎一切都有值。`if` 是表达式,`match` 是表达式,代码块也是表达式。没有"语句"与"表达式"的割裂,也就没有"这个函数早期返回了 `undefined`"这类陷阱。

```1y
let status =
  if score >= 90 then "A"
  else if score >= 60 then "B"
  else "C"
```

表达式导向让程序更接近数学:程序就是"计算",而非"指令序列"。这也使得重构更加安全——任何一段代码都可以被提取为函数,因为它本就有值。

### 数值统一(Numerical Unification)

1y 只有两种数值类型:`Int`(任意精度整数)与 `Decimal`(任意精度十进制小数)。算术运算会自动提升:`Int + Decimal` 得到 `Decimal`;除法不能整除时自动提升为 `Decimal`。程序员永远不必在 `int`、`long`、`float`、`double`、`BigInteger`、`BigDecimal` 之间做选择——选择由语言正确地代劳。

```1y
let a = factorial(500)        # 一个 1135 位的整数,毫无问题
let b = 10 :pow 100           # 1 后面 100 个 0
let c = 1 / 3                 # 0.3333...(Decimal),而非 0
```

关于数值统一,详见[数值统一](./numerical-unification)一章。

### 并发即一等公民(Concurrency as First-class)

并发不是附加的库,而是语言的核心。Actor 与 STM 都内建于语言,拥有专属语法:

```1y
let counter = spawn Counter.init(0)
counter ! Incr                       # 发后即忘
let current = counter ? Get          # 请求/回复
```

并发原语不是"高级特性",而是每个 1y 程序员日常使用的工具。详见[并发模型](./concurrency-model)。

## 这些原则如何塑造语言

这四条原则彼此 reinforcing(相互强化)。不可变性让 Actor 之间可以安全地传递引用,无需拷贝;表达式导向让 STM 事务体天然就是一个"计算值",回滚语义清晰;数值统一消除了类型转换带来的边界 bug,让并发数值计算更可靠。

它们共同作用的结果是:**1y 的程序读起来像一份规格说明**。代码描述"是什么",而不是"怎么做"。当你阅读一段 1y 代码时,你很少需要在大脑里模拟一个逐步变化的状态机——因为没有变化,只有值的流动。

## 与其他语言的对比

| 维度 | 1y | Java/Go | Rust | Haskell |
|------|-----|---------|------|---------|
| 整数 | 任意精度 | 定长(易溢出) | 定长 | 任意精度 |
| 可变性 | 默认不可变 | 默认可变 | 默认不可变 | 默认不可变 |
| 并发 | Actor + STM | 线程/锁、channel | 线程、async | STM、async |
| 函数着色 | 无 | 无 | 有(async) | 有 |
| 心智负担 | 低 | 中(锁) | 高(借用) | 中 |

1y 与 Haskell 在价值观上最为接近,但刻意避开了 Haskell 的类型系统复杂度与惰性求值的意外开销。1y 是严格求值的,类型系统保持轻量,目的是让"函数式的正确性"能被更多程序员享受。

## 麻雀虽小,五脏俱全

1y 的语法表很小:`let`、`fn`、`match`、`if`、`actor`、`atomically`、模块系统、FFI——几乎可以一张纸写完。但"小"不等于"弱":

- 任意精度数值,足以支撑金融与科学计算;
- 持久化数据结构,足以支撑复杂的数据变换;
- 模式匹配,足以表达精密的分支逻辑;
- Actor + STM,足以构建高并发系统;
- FFI,足以复用 Rust 生态的全部力量。

这种"小而全"的哲学,是 1y 对"语言应当克制"这一信念的实践。一门语言的力量,不在于它提供了多少关键字,而在于它用最少的原语组合出了多大的表达空间。1y 选择做一只五脏俱全的麻雀:每一处设计都为某个真实问题而存在,没有为炫技而生的特性。

当你读完这份文档,你会发现 1y 没有宏、没有类型类、没有 effect 系统、没有 async。这些"缺席"都是有意为之——它们每一个都对应着一种复杂度的引入,而 1y 相信,用更简单的原语同样能达到目标。接下来的章节将逐一展开这些抉择背后的理由。
