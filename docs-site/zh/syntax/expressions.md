# 表达式与运算符

1y 是一门表达式导向的语言:几乎一切都有值。`if` 是表达式,`match` 是表达式,代码块也是表达式。本章覆盖 1y 的全部运算形式——从算术到管道,从索引到字段访问,以及它们各自的语义细节。理解这些规则后,你就能读懂并写出地道的 1y 代码。

## 算术运算

1y 提供五种算术运算符:`+` `-` `*` `/` `%`。它们既适用于 `Int` 也适用于 `Decimal`,并遵循一套自动提升规则。

### 基本运算

```1y
let a = 7 + 2;        // 9
let b = 7 - 2;        // 5
let c = 7 * 2;        // 14
let d = 7 % 2;        // 1
```

### 除法与自动提升

除法 `/` 的行为值得特别留意:**当两个整数能整除时,结果保持 `Int`;不能整除时,自动提升为 `Decimal`**。这意味着你永远不会因整数除法而丢失小数部分。

```1y
let e = 6 / 2;        // 3,Int(能整除)
let f = 7 / 2;        // 3.5,Decimal(不能整除,自动提升)
let g = 1 / 3;        // 0.3333...,Decimal
```

### 混合运算的提升

当 `Int` 与 `Decimal` 一起运算时,`Int` 会自动提升为 `Decimal`,结果也是 `Decimal`。你不必手动转换类型——语言会替你做正确的选择。

```1y
let h = 1 + 0.5;       // 1.5,Decimal
let i = 2 * 3.14;       // 6.28,Decimal
let j = 10 - 0.1;       // 9.9,Decimal
```

这套规则的核心理念是:**程序员不应为数值类型的宽度操心**。无论你混合使用 `Int` 与 `Decimal`,还是做除法,语言都会给出数学上正确的结果。

## 比较运算

比较运算符 `<` `<=` `>` `>=` `==` `!=` 返回 `Bool`。它们对数值、字符串、布尔值都适用:

```1y
let lt = 3 < 5;          // true
let ge = 3.14 >= 3.14;   // true
let eq = "a" == "a";     // true
let ne = true != false;  // true
```

数值比较跨越 `Int` 与 `Decimal` 同样有效:`1 == 1.0` 为 `true`。

## 逻辑运算

逻辑运算使用单词关键字 `and`、`or`、`not`,而非符号 `&&`、`||`、`!`。它们都是**短路求值**的:

```1y
let ok = true and false;        // false
let yes = true or false;        // true
let no = not true;              // false

// 短路:第二个操作数只在需要时才求值
let x = nil;
let safe = x != nil and x > 0;  // false,不会触发对 nil 的比较错误
```

短路求值让你可以安全地写出"先判断再访问"的链式条件,无需额外的 `if` 嵌套。

## 管道运算符 `|>`

管道是 1y 中最具表现力的运算符之一,它让"数据流"式编程变得自然。`x |> f` 等价于 `f(x)`——把左侧的值"灌入"右侧的函数。

### 单参数管道

```1y
let double = fn(x) { x * 2 };
let inc = fn(x) { x + 1 };

// 5 |> double |> inc  ==  inc(double(5)) == 11
let result = 5 |> double |> inc;
```

链式管道读起来像一条流水线:值从左向右流过一连串变换。这与嵌套的函数调用 `inc(double(5))` 在数学上等价,但可读性天差地别。

### 带额外参数的管道

当管道右侧的函数需要额外参数时,写成 `x |> f(a)`——它等价于 `f(a, x)`,即**把管道左侧的值追加到参数列表末尾**:

```1y
let add = fn(a, b) { a + b };
// 5 |> add(10)  ==  add(10, 5) == 15
let sum = 5 |> add(10);
```

这一约定让管道能与 `map`、`filter`、`fold` 等多参数高阶函数无缝配合:

```1y
let nums = [1, 2, 3, 4, 5, 6];
let pipeline = nums
    |> filter(fn(x) { x % 2 == 0 })
    |> map(fn(x) { x * 10 })
    |> fold(0, fn(a, b) { a + b });
// 120  (20 + 40 + 60)
```

## 索引访问

用方括号 `[]` 对 `Vec` 与 `Map` 做索引访问:

```1y
let xs = [10, 20, 30];
let first = xs[0];          // 10

let m = {"a": 1, "b": 2};
let val = m["a"];            // 1
```

对 `Vec` 而言,索引是 `Int`;对 `Map` 而言,键可以是任意 `Value`。索引访问同样支持赋值形式 `xs[0] = 99`,这会返回一个修改后的新集合。

## 字段访问

点号 `.` 有两种含义,取决于左侧是结构体还是 `Map`。

### 结构体字段

对 `Struct` 值,`s.x` 读取名为 `x` 的字段:

```1y
type Point = { x: Int, y: Int }
let p = Point({ x: 3, y: 4 });
let px = p.x;               // 3
p.x = 42;                    // 赋值,返回新结构体
```

### Map 的字段简写

对 `Map` 值,`m.key` 是 `get(m, "key")` 的简写——只要键是字符串,就能用点号而非方括号:

```1y
let m = { name: "alice", age: 30 };
let name = m.name;          // 等价于 get(m, "name"),值为 "alice"
let age = m.age;            // 30
```

这让从 `Map`(或经 JSON 解析得到的对象)中取值如同访问结构体字段一样自然。

## 方法调用

`recv.method(args)` 是一种语法糖:它从接收者 `recv` 上取出名为 `method` 的函数值,然后调用它。关键细节在于**接收者如何被传递**取决于 `recv` 的类型。

### 值接收者:接收者作为第一个参数

当 `recv` 是普通值(结构体、`Vec`、`Map` 等)时,接收者本身会作为函数的**第一个参数**传入:

```1y
// 等价于 push(vec, 4)
let vec = [1, 2, 3];
vec.push(4);                 // 返回 [1, 2, 3, 4]

// 等价于 get(m, "key")
let m = { key: "v" };
m.get("key");                // "v"
```

### 模块接收者:接收者不传递

**例外**:当 `recv` 是一个 `Module` 时,接收者**不会**被传入——模块方法本身就是独立的函数,点号只是命名空间限定:

```1y
import io;
// io.read_to_string("file") 直接调用 read_to_string("file")
// 不会把 io 本身作为参数传入
let content = io.read_to_string("file.txt");
```

这一区分非常重要:模块是函数的容器,而非函数的第一个参数。记住这条规则,你就能正确判断一个方法调用到底展开成什么。

## 字符串插值

在双引号或三引号字符串中,`{expr}` 会被求值并替换为结果的字符串形式:

```1y
let name = "world";
let n = 42;
let msg = "hello, {name}! answer = {n}";   // hello, world! answer = 42

// 插值里可以是任意表达式
let xs = [1, 2, 3];
let s = "sum = {fold(xs, 0, fn(a, b) { a + b })}";   // sum = 6
```

若字符串中需要字面的花括号,用 `\{` 与 `\}` 转义。字符串拼接也可以用 `+`,但插值通常更清晰:

```1y
let a = "hello" + " " + "world";    // 拼接
let b = "hello, {name}!";            // 插值,更推荐
```

## 赋值不是表达式

最后需要强调:在 1y 中,**赋值是语句,不是表达式**。`x = 5` 不能作为表达式的一部分使用(例如不能写 `if (x = 5) { ... }`)。这一设计避免了把 `=` 误写成 `==` 导致的经典 bug。复合赋值 `+=`、`-=` 等同样是语句。详见[语句](./statements)一章。
