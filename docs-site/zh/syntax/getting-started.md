# 快速开始

欢迎来到 1y 的世界。1y(读作 "one-why")是一门流式、并发、函数式编程语言,使用 Rust 实现。它具备任意精度数值、持久化数据结构、模式匹配、Actor 并发、软件事务内存(STM)、模块系统与 FFI。本章将带你从零开始:安装编译器、运行第一个程序、熟悉 CLI 工具,并跑通随仓库附带的所有示例。

## 从源码构建

1y 目前通过源码分发。你需要在系统上安装 Rust 工具链(`rustc` 与 `cargo`)。如果你的机器上还没有 Rust,请按 [rustup.rs](https://rustup.rs/) 的指引安装稳定版工具链。

确认 `cargo` 可用后,克隆仓库并构建:

```sh
git clone https://github.com/onely/1y.git
cd 1y
cargo build --release
```

构建完成后,可执行文件位于 `target/release/1y`(Windows 上是 `target\release\1y.exe`)。建议把它所在的目录加入 `PATH`,这样你就能在任何地方直接敲 `1y` 来调用它。

如果你只想快速验证编译器能否工作,也可以不切到 release 构建而使用 debug 构建:

```sh
cargo build            # 产出 target/debug/1y
```

debug 构建更快,但运行性能不如 release 构建。对于学习与试玩,debug 构建已经足够。

## Hello, World

依照传统,我们的第一个程序只输出一行文字。新建一个文件 `hello.1y`,写入:

```1y
println("Hello, World!")
```

然后用 CLI 运行它:

```sh
1y run hello.1y
```

你应该会看到终端打印出 `Hello, World!`。就是这么简单——1y 程序从文件的第一条语句开始顺序执行,不需要 `main` 函数,也不需要任何样板代码。

## 命令行工具

1y 的 CLI 提供三个核心子命令,分别对应"运行"、"查看 AST"和"查看 Token"三种用途。它们在你调试语法或理解语言内部机制时格外有用。

### `1y run` — 执行程序

最常用的命令。它读取一个 `.1y` 文件,解析、求值并执行其中的语句:

```sh
1y run path/to/program.1y
```

程序的输出由其中的 `println`、`print` 等函数产生。任何未捕获的异常会以错误信息的形式打印到 stderr,并以非零退出码结束进程。

### `1y parse` — 打印 AST

把源码解析为抽象语法树(AST)并打印出来,但不执行。当你想确认某段语法被解析成什么结构时非常有用:

```sh
1y parse hello.1y
```

例如,对于 `let x = 1 + 2;`,你会看到对应的 `Let` 节点,内嵌一个 `Binary` 表达式。这是理解 1y 语法结构的绝佳入口。

### `1y tokens` — 打印词法 Token

只做词法分析,把源码拆解为 Token 流并逐个打印。当你要排查某个字面量或运算符是否被正确识别时,这个命令能立刻给出答案:

```sh
1y tokens hello.1y
```

例如,`"hello {name}"` 会被拆成字符串起始、插值表达式、字符串结束等多个 Token,让你直观地看到插值是如何被词法器处理的。

## 一个 .1y 文件的结构

一个典型的 1y 源文件由若干顶层语句组成,从上到下顺序执行。下面是一个结构完整的示例,涵盖了导入、绑定、函数定义与表达式语句:

```1y
// 导入标准库模块
import io;

// 顶层绑定:绑定一个不可变的值
let greeting = "hello";

// 函数声明
fn add(a, b) {
    a + b
}

// 表达式语句:调用函数并打印结果
println(greeting + ", sum = " + str(add(3, 4)));
```

几个要点:

- **没有入口函数**:文件本身就是入口,从第一条语句开始执行。
- **顶层绑定即导出**:当这个文件被作为模块导入时,它的顶层 `let` 与 `fn` 绑定就成为该模块的导出。
- **分号**:语句通常以分号结尾;但在块表达式的末尾或文件末尾,分号可以省略。
- **不可变优先**:`let` 绑定的值默认不可变。需要重新赋值时,直接用 `=` 重新绑定同名变量。

## 运行示例程序

仓库的 `examples/` 目录包含一组按开发阶段组织的示例程序,从 `phase0` 到 `phase4.6`,每一阶段演示一组特定的语言特性。建议按顺序运行它们,你会看到 1y 的能力是如何一步步叠加的。

```sh
1y run examples/phase1.1y      // 任意精度算术、持久化集合、模式匹配、闭包、异常、管道
1y run examples/phase2.1y      // Actor 模型:spawn、消息传递、状态隔离、状态机
1y run examples/phase3.1y      // 软件事务内存:shared、transact、回滚、retry、嵌套事务
1y run examples/phase3.5.1y    // 循环、复合赋值、高阶函数、字符串与数学函数
1y run examples/phase4.1y      // 模块系统:import、alias、lazy import 与标准库
1y run examples/phase4.5.1y    // crypto、tls 与 ffi 模块概览
1y run examples/phase4.6.1y    // 真实 FFI:加载动态库、调用 C 函数
```

以 `phase1.1y` 为例,它展示了 1y 的"前端"能力——任意精度算术(100 的阶乘毫无压力)、持久化集合的结构共享、闭包与高阶函数、自定义类型(结构体与枚举)、带守卫与 Or 模式的模式匹配、异常处理,以及管道运算符。运行它,你会得到一连串带注释的输出,每一节都对应一个语言特性。
