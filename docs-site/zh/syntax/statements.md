# 语句

1y 是表达式导向的——大多数构造都有值。但程序仍需要"语句"这一层来组织绑定、声明与导入。本章覆盖 1y 的全部语句形式:从最基础的 `let` 绑定,到函数声明、自定义类型声明,再到模块导入。理解每种语句的语义后,你就能把表达式组合成完整、可运行的程序。

## `let` 绑定

`let` 用于创建一个**新的**变量绑定。绑定默认不可变——一旦绑定,该名字在当前作用域内指向一个固定的值:

```1y
let x = 42;
let name = "alice";
let xs = [1, 2, 3];
```

`let` 会在当前作用域内引入新名字。若同名变量已存在于外层作用域,`let` 会在当前作用域内**遮蔽**(shadow)它,而非修改外层变量:

```1y
let n = 1;
{
    let n = 99;        // 遮蔽外层的 n,仅在本块内有效
    println(n);        // 99
};
println(n);            // 1,外层 n 未受影响
```

## 重新赋值 `=`

与 `let` 不同,`=` 用于**重新赋值一个已存在的变量**。它不创建新绑定,而是修改当前作用域里已有的同名变量:

```1y
let x = 1;
x = 99;                 // 重新赋值,不是 let
println(x);             // 99
```

### `let` 与 `=` 的区别

这是初学者最易混淆的一点,务必牢记:

- **`let x = expr;`** ——声明一个**新**变量。若同名变量已存在,则遮蔽它。
- **`x = expr;`** ——对**已存在**的变量重新赋值。若该名字尚未声明,会报错。

```1y
let a = 1;       // 声明新变量 a
a = 2;            // 重新赋值已有变量 a
let a = 3;        // 再次声明,遮蔽前一个 a
```

由于值默认不可变,`=` 重新赋值本质上是"让这个名字指向另一个值",而非原地修改内存。对持久化集合而言,这尤为重要。

## 复合赋值

为方便书写,1y 提供五种复合赋值运算符,它们都是"读取—运算—重新赋值"的简写:

```1y
let n = 10;
n += 5;            // 等价于 n = n + 5    → 15
n -= 3;            // 等价于 n = n - 3    → 12
n *= 2;            // 等价于 n = n * 2    → 24
n /= 4;            // 等价于 n = n / 4    → 6
n %= 4;            // 等价于 n = n % 4    → 2
```

复合赋值同样适用于集合上的索引与字段位置:

```1y
let xs = [1, 2, 3];
xs[0] += 10;        // xs[0] 从 1 变成 11

let m = { count: 0 };
m.count += 1;       // count 从 0 变成 1
```

## 函数声明 `fn`

`fn` 声明一个函数,并将其绑定到给定名字。函数是一等值,可以作为参数传递、作为返回值:

```1y
fn add(a, b) {
    a + b
}

fn factorial(n) -> Int {
    if n <= 1 { 1 } else { n * factorial(n - 1) }
}
```

函数体是一个块表达式,其末尾表达式的值即为返回值——**无需 `return` 关键字**(虽然 1y 支持 `return`,但大多数情况下不需要)。

### 可选的类型注解

参数与返回类型可以附带类型注解,但它们目前主要用于文档目的,不影响运行时行为:

```1y
fn greet(name: Str) -> Str {
    "hello, " + name
}
```

### 匿名函数(λ)

除了命名函数,你也可以用 `fn(params) { body }` 直接写匿名函数,把它绑定到 `let` 或作为参数传递:

```1y
let double = fn(x) { x * 2 };
let add = fn(a, b) { a + b };

// 作为高阶函数参数
let xs = [1, 2, 3];
let squares = map(xs, fn(x) { x * x });
```

闭包以引用方式捕获定义环境,因此内部函数可以引用外层变量:

```1y
fn make_adder(x) {
    fn(y) { x + y }       // 捕获外层的 x
}

let add10 = make_adder(10);
add10(5);                  // 15
```

## 结构体类型声明 `type`

`type` 声明一个结构体类型——具名字段的记录:

```1y
type Point = { x: Int, y: Int };

let p = Point({ x: 3, y: 4 });
println(p.x);              // 3
```

声明之后,类型名 `Point` 本身就是一个构造函数:`Point({...})` 会创建一个 `Struct` 值。字段通过点号 `p.x` 访问,通过 `p.x = ...` 赋值。结构体也支持在 `match` 中解构,详见[模式匹配](./pattern-matching)。

## 枚举声明 `enum`

`enum` 声明一组带名构造子(变体),每个变体可携带零或多个参数。它用于表达"和类型":

```1y
enum Shape {
    Circle(Int),
    Rect(Int, Int)
}

enum Result {
    Ok(Int),
    Err(String)
}
```

声明之后,`Circle`、`Rect`、`Ok`、`Err` 都是可用的构造函数,调用它们会产生 `Variant` 值:

```1y
let c = Circle(5);
let r = Rect(3, 4);
let ok = Ok(42);

let area = match c {
    Circle(r) => r * r,
    Rect(w, h) => w * h
};
```

不带参数的变体(如 `None`)本身就是值,无需调用。枚举与模式匹配结合,是 1y 表达分支逻辑的主要手段。

## 模块导入 `import`

`import` 把一个模块绑定到当前作用域。1y 支持三种导入形式:

```1y
import io;                  // 默认绑定名 = 路径末段
import io as fs;            // 别名:绑定名为 fs
lazy import heavy_lib;      // 延迟加载:首次使用时才真正加载
```

### 即时导入与别名

`import io;` 会立即加载 `io` 模块,并以 `io` 为名绑定它。`as` 可改变绑定名:

```1y
import io as fs;            // 之后用 fs.read_to_string(...) 调用
import process as proc;     // 之后用 proc.pid() 调用
```

### 延迟导入 `lazy import`

`lazy import` 只绑定一个占位符,模块在被**首次求值**时才真正加载。这对可选依赖或希望加快启动时间的场景很有用:

```1y
lazy import random as rnd;
// 直到此处访问 rnd 才触发加载
println(rnd.int(10));
```

### 模块路径解析

- **标准库模块**:`io`、`env`、`json`、`process`、`random`、`socket`、`crypto`、`tls`、`serial`、`ffi`。
- **文件模块**:路径 `a.b.c` 会解析为 `<入口目录>/a/b/c.1y`。一个 `.1y` 文件的顶层绑定即为其导出。
- 模块按规范路径缓存;循环导入会触发错误。

## 表达式语句

任何表达式后跟分号 `;` 就构成一条语句——这类语句通常用于产生副作用(如打印),而忽略其值:

```1y
println("hello");           // 调用 println,忽略返回值(nil)
x + 1;                       // 求值但丢弃结果(罕见,通常无意义)
counter ! Inc(1);           // 向 Actor 发送消息
```

块表达式 `{ ... }` 同样可以作为语句,其内部最后一个表达式的值即为块的值。这使得任意一段代码都可以被提取为块,而无需改变语义:

```1y
let result = {
    let a = 10;
    let b = 20;
    a + b
};                          // result 为 30
```
