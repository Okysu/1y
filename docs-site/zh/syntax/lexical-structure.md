# 词法结构

任何编程语言都从字符开始。在 1y 能理解你的代码之前,词法器(lexer)会把源码字符流切分成一个个 Token——注释、标识符、字面量、运算符与标点。本章逐一说明 1y 的词法规则,帮助你理解哪些字符序列是合法的,以及它们分别会被解析成什么。

## 注释

1y 提供两种注释。它们在词法阶段被丢弃,不影响程序语义。

### 行注释

以 `//` 开头,延续到行尾。适合简短的行内说明:

```1y
// 这是一行注释
let x = 1;   // 行尾注释也是合法的
```

### 块注释

以 `/*` 开头、`*/` 结尾,可以跨越多行。**块注释是可嵌套的**,这一点与 C/C++/Java 不同——你不必担心嵌套的注释会提前结束:

```1y
/* 这是一段
   多行注释 */
let y = 2;

/* 外层注释
   /* 内层注释,合法嵌套 */
   外层继续 */
let z = 3;
```

嵌套注释在临时屏蔽大段代码(其中本身就含块注释)时特别方便。

## 标识符

标识符用来命名变量、函数、类型与模块。规则很直观:

- 以**字母**或下划线 `_` 开头;
- 后续字符为字母、数字或下划线。

```1y
let name = "alice";
let _count = 0;
let total2 = 42;
let __internal = nil;
```

标识符区分大小写:`Name` 与 `name` 是两个不同的标识符。注意,以数字开头的字符序列会被解析为数字字面量,因此标识符不能以数字开头。

## 数字字面量

1y 只有两种数值类型——`Int`(任意精度整数)与 `Decimal`(任意精度十进制小数),词法器会根据是否包含小数点或指数来区分它们。

### 整数(Int)

由一串数字组成,可以插入下划线作为视觉分隔(不影响数值):

```1y
let a = 42;
let b = 1_000_000;          // 等于 1000000
let huge = 170141183460469231731687303715884105727;   // 任意精度,不会溢出
```

下划线可以出现在数字之间任意位置,这让大数字更易读:`1_000_000` 与 `1000000` 完全等价。

### 小数(Decimal)

包含小数点或指数标记的数字字面量被解析为 `Decimal`:

```1y
let pi = 3.14;
let precise = 3.141592653589793238462643383279502884197;
let exponent = 0.5e10;        // 指数表示
let big_exp = 1.5e10;
```

`Decimal` 同样是任意精度,适合金融与科学计算。算术运算的自动提升规则(如 `Int + Decimal` 得到 `Decimal`)将在[表达式](./expressions)一章详述。

## 字符串字面量

1y 的字符串是 UTF-8 编码的,支持插值与多行书写。

### 双引号字符串

最常见的形式,用双引号包裹:

```1y
let s = "hello, world";
let empty = "";
```

### 三引号字符串

用三个双引号 `"""` 包裹,可以跨越多行,适合书写大段文本:

```1y
let poem = """
床前明月光,
疑是地上霜。
""";
```

三引号字符串会保留内部的换行与缩进,常用于模板、文档或 JSON 片段。

### 字符串插值

在双引号或三引号字符串中,`{expr}` 会被替换为表达式的值——这是 1y 最常用的字符串构造方式:

```1y
let name = "world";
let n = 42;
println("hello, {name}! answer = {n}");

// 插值里可以是任意表达式
let xs = [1, 2, 3];
println("sum = {fold(xs, 0, fn(a, b) { a + b })}");
```

### 转义与字面花括号

字符串支持常见的转义序列:`\n`(换行)、`\t`(制表符)、`\\`(反斜杠)、`\"`(双引号)。**当字符串中需要字面的花括号字符时**,使用 `\{` 与 `\}` 转义,否则它们会被当作插值:

```1y
let json_text = "\{\"name\": \"alice\"\}";   // 字面的 {"name": "alice"}
let path = "C:\\Users\\me";                  // 字面的反斜杠
```

这一点在书写 JSON、模板字符串等含大量花括号的文本时至关重要。

## 关键字

下列单词在 1y 中是保留字,不能作为标识符使用:

| 类别 | 关键字 |
|------|--------|
| 绑定与定义 | `let` `fn` `type` `enum` `struct` |
| 控制流 | `if` `else` `match` `while` `for` `in` `loop` `break` `continue` `return` |
| 异常 | `raise` `try` `rescue` `ensure` |
| 并发 | `actor` `on` `spawn` `reply` `receive` `shared` `transact` `retry` |
| 模块 | `import` `lazy` `as` |
| 逻辑 | `and` `or` `not` |
| 字面量 | `true` `false` `nil` |

如果你确实需要把某个关键字当作名字(例如调用一个名为 `type` 的外部函数),通常通过模块字段访问的形式 `m.type` 来绕过——这会在[语句](./statements)与[模块系统](./modules)中讨论。

## 运算符与标点

1y 使用以下运算符与标点符号:

| 类别 | 符号 |
|------|------|
| 算术 | `+` `-` `*` `/` `%` |
| 比较 | `<` `<=` `>` `>=` `==` `!=` |
| 赋值 | `=` `+=` `-=` `*=` `/=` `%=` |
| 管道 | `\|>` |
| 逻辑 | `and` `or` `not`(单词,非符号) |
| 索引/字段 | `.` `[` `]` |
| 消息 | `!` `?` |
| 解引用 | `*`(用于读写 `shared` 单元) |
| 分隔 | `(` `)` `{` `}` `,` `;` `:` `->` `=>` `\|` `..` |

其中 `!` 用于向 Actor 发送"发后即忘"消息,`?` 用于同步请求/回复;`*` 在 `shared` 上下文中表示读写事务单元。这些会在后续并发章节展开。
